Vision at the Edge

Hailo Vision turns your Pi fleet into a powerful computer vision platform. No GPU servers. No cloud APIs. Just dedicated AI silicon doing what it does best.

👁

Object Detection

YOLOv8 and SSD models running at 30 FPS on Hailo-8. Detect and classify 80+ object categories in real-time video streams. Custom model training for domain-specific objects. Batch processing for offline video analysis.

👤

Face Recognition

ArcFace and RetinaFace models for face detection and recognition. Build and manage face databases locally. No biometric data ever leaves your device. Configurable matching thresholds for security vs. convenience trade-offs.

🎥

Video Analytics

Multi-camera stream processing with object tracking across frames. Heatmaps, people counting, dwell time analysis, path tracking. All computed on the Hailo-8 without sending a single frame to any server.

🔒

Privacy-First Design

All inference happens on-device. Video frames are processed and discarded — never stored unless explicitly configured. Audit logs track every detection event. GDPR-ready by architecture, not by policy.

🔌

Model Pipeline

Chain multiple models in sequence: detect, crop, classify, act. Pre-processing (resize, normalize) and post-processing (NMS, filtering) built into the pipeline. Hot-swap models without restarting the pipeline.

📡

Event-Driven Actions

Trigger webhooks, MQTT messages, or RoadFlow workflows based on detection events. "When person detected in zone A after 10pm, alert security and record 30s clip." Complex event logic with temporal conditions.

Purpose-Built
Hardware

The Hailo-8 is a dedicated AI processor designed from the ground up for edge inference. Unlike GPU-based solutions that waste power on general compute, the Hailo-8 dataflow architecture delivers maximum TOPS per watt.

Two Hailo-8 chips in the BlackRoad fleet — one on Cecilia, one on Octavia — provide a combined 52 TOPS of AI compute. That's enough to run multiple YOLOv8 models simultaneously across dozens of camera feeds.

The M.2 form factor means these chips integrate directly into any Raspberry Pi 5 with the AI HAT+. No external GPUs, no power-hungry servers, no noise.

Processing Pipeline

From camera frame to actionable event in under 33 milliseconds.

01 — CAPTURE

Video Input

RTSP, USB, CSI camera streams. Multi-camera multiplexing. Configurable resolution and frame rate. Hardware-accelerated decode via V4L2.

02 — PROCESS

Hailo Inference

Frame sent to Hailo-8 via PCIe. Model executes on dedicated silicon. Pre/post-processing on CPU. Multiple models can share the accelerator.

03 — ACT

Event Dispatch

Detection results trigger events via MQTT, webhooks, or RoadFlow. Metadata logged to SQLite/PostgreSQL. Annotated frames optionally saved or streamed.

# Hailo Vision — deploy a detection pipeline

# Install
pip install hailo-vision

# Configure camera
hailo-vision config add-camera \
  --name "front-door" \
  --url "rtsp://192.168.4.10/stream1" \
  --model "yolov8s" \
  --fps 30

# Add detection zone
hailo-vision zone add \
  --camera "front-door" \
  --name "entrance" \
  --polygon "100,100 500,100 500,400 100,400"

# Set alert rule
hailo-vision alert add \
  --zone "entrance" \
  --class "person" \
  --after "22:00" \
  --action "webhook:https://roadflow.local/alert"

# Start pipeline
hailo-vision start --daemon

Deploy in
Minutes

The Hailo Vision CLI makes it simple to configure cameras, define detection zones, set alert rules, and start processing. No YAML manifests. No container orchestration. Just commands that do what they say.

Models are pre-compiled for Hailo-8 and cached locally. First run downloads the model once. After that, everything runs offline. Update models with a single command when better versions are available.
